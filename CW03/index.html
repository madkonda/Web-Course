Exercise 2: Debugging CSS Code

Task: identify and fix 3 common CSS mistakes.

Original buggy CSS (given)

/* Mistake 1 */
body {
    font-family: "Times New Roman";
    margin-left: 20%;
    margin-right: 20%;
    text-align: justify;
    background: ivory;
    color: red
}

/* Mistake 2 */
h1, h2, h3 {
    color: #ff0000;
    font-size: 2.5em;
    margin: 20px 0px 20px 0px;
}

/* Mistake 3 */
.container {
    width: 100%;
    padding: 20px;
    box-sizing: border-box
}

a {
    color: #0000FF;
    text-decoration: underline;
}

Answers (3 mistakes) + fixes

  1. Mistake 1: font-family has no fallback font.
    Fix: add a generic fallback like serif.
  2. Mistake 2: missing semicolon after color: red in body.
    Fix: color: red;
  3. Mistake 3: missing semicolon after box-sizing: border-box.
    Fix: box-sizing: border-box;

Corrected CSS

body {
  font-family: "Times New Roman", serif;
  margin-left: 20%;
  margin-right: 20%;
  text-align: justify;
  background: ivory;
  color: red;
}

h1, h2, h3 {
  color: #ff0000;
  font-size: 2.5em;
  margin: 20px 0;
}

.container {
  width: 100%;
  padding: 20px;
  box-sizing: border-box;
}

a {
  color: #0000ff;
  text-decoration: underline;
}

Exercise 4: External Stylesheet (lyrics.css)

Goal: move lyrics styles into an external stylesheet.

The lyrics styling is stored in css/lyrics.css (external file). It is linked from this page using: 

<link rel="stylesheet" href="css/lyrics.css">

To avoid affecting the entire page, the selectors in lyrics.css are scoped to the .lyrics-page container used in Exercise 3.
